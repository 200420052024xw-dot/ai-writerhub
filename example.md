# UI 分析

## 素材来源

前端示例图位于根目录 `fronted_picture` 文件夹，包含四个页面：

- `编辑器.png`
- `翻译.png`
- `格式整理.png`


## 整体布局

四张示例图保持统一的信息架构：

- 左侧固定导航栏：展示产品标识和一级功能入口。
- 顶部文档栏：展示菜单按钮、当前文档标题、保存状态、搜索框、用户区。
- 中央主工作区：承载当前功能的核心操作。
- 右侧辅助栏：提供 AI 助手、增强选项、导出操作、检索结果或引用片段。
- 底部状态栏：展示字数、字符数、阅读时长、Markdown 状态或引擎状态。

## 导航结构

左侧导航包含：

- 编辑器
- 翻译
- 格式整理
- 文档问答
- 导出中心
- 设置

导航高亮状态使用浅蓝背景和左侧蓝色指示条，当前页面文字与图标为蓝色。

## 编辑器页面

### 页面结构

- 顶部工具区：
  - 编辑视图
  - 预览视图
  - 分屏
  - Markdown 检测状态
  - 复制 Markdown
  - 复制纯文本
  - 更多操作
  - 导出
- 中央区域：
  - 左侧 Markdown 编辑器
  - 右侧 Markdown 预览
  - 中间可拖动分隔条
- 右侧面板：
  - AI 助手
  - 常用 AI 操作按钮
  - 问题或指令输入框

### 交互重点

- 分屏编辑是主要模式。
- 编辑区需要显示行号。
- Markdown 预览应实时同步。
- AI 操作应围绕当前文档内容，而不是独立聊天。

## 翻译页面

### 页面结构

- 顶部切换：
  - 中英互译
  - 文档翻译
- 翻译控制区：
  - 中 -> 英
  - 英 -> 中
  - 切换方向
  - 开始翻译
- 展示模式：
  - 分屏对照
  - 段落交替
  - 句对句对照
- 中央区域：
  - 左侧中文原文
  - 右侧英文译文
- 右侧面板：
  - AI 增强选项
  - 术语表 / 记忆库
- 底部区域：
  - 其他结果视图预览

### 交互重点

- 翻译结果需要便于校对。
- 展示方式切换应保留同一批翻译内容。
- 术语表需要影响后续翻译策略。

## 格式整理页面

### 页面结构

- 左侧配置区：
  - 自然语言格式输入框
  - 智能解析
  - 应用格式
  - 重置
  - 解析出的格式配置
  - 格式化范围
- 中央预览区：
  - 页面翻页
  - 缩放比例
  - 页面视图
  - 文档排版预览
- 右侧操作区：
  - 导出 Word
  - 导出 PDF
  - 打印预览
  - 格式变更记录
  - 格式建议

### 交互重点

- 自然语言输入只是入口，最终应落到结构化配置。
- 预览区应接近真实 Word / PDF 页面效果。
- 导出前应能确认格式已经应用。

## 文档问答页面

### 页面结构

- 左侧文档库：
  - 上传文档
  - 文档列表
  - 解析状态
  - 选择参与问答的文档
- 中央问答区：
  - 文档范围筛选
  - 排序方式
  - 用户问题
  - AI 回答
  - 回答内引用标签
  - 继续提问输入框
- 右侧检索区：
  - 检索结果 Top 5
  - 引用片段
  - 查看原文

### 交互重点

- 用户需要明确当前问题基于哪些文档。
- 回答必须带来源，而不是只给结论。
- 文档解析和索引状态必须可见。

## 视觉风格

- 主色：蓝色。
- 背景：浅灰白工作台。
- 组件风格：细边框、轻阴影、圆角较小。
- 信息密度：偏工具型，不做营销式大图区。
- 图标：优先使用常见线性图标。
- 按钮层级：主要操作使用实心蓝色，次要操作使用描边或浅底。
