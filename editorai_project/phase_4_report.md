# 第四阶段验收报告

## 完成内容

本阶段完成中英互译流程，翻译页从静态 UI 升级为可输入、可配置、可调用 FastAPI mock 翻译接口的功能页。

### 前端

- 重写 `frontend/src/pages/TranslatePage.tsx`。
- 支持翻译方向：
  - 中 -> 英
  - 英 -> 中
- 支持三种展示方式：
  - 分屏对照
  - 段落交替
  - 句对句对照
- 支持编辑原文输入。
- 支持点击“开始翻译”调用后端接口。
- 支持翻译加载状态。
- 支持复制译文。
- 支持翻译增强选项：
  - 学术风格
  - 商务风格
  - 口语自然
  - 术语统一
  - 保留专有名词
- 保留术语表 / 记忆库展示。

### 后端

- 新增 `POST /api/translate`。
- 新增翻译请求和响应模型。
- 新增 mock 翻译服务。
- 响应中返回：
  - 原文
  - 译文
  - 翻译方向
  - 展示模式
  - 段落对照数据
  - 句子对照数据
  - 已应用的翻译选项

## 验收标准

- 翻译页可以输入原文。
- 可以切换中英翻译方向。
- 可以切换分屏、段落交替、句对句对照展示。
- 点击开始翻译后，前端调用 FastAPI 接口并展示结果。
- 翻译增强选项会进入请求参数。
- 前端构建通过。
- 后端健康检查和翻译接口可访问。

## 验收结果

- `npm run build`：通过。
- FastAPI 路由注册检查：通过，存在 `/api/translate`。
- `GET http://127.0.0.1:8000/api/health`：通过，返回 `status=ok`。
- `POST http://127.0.0.1:8000/api/translate`：通过，返回翻译响应结构。
- 翻译服务函数验证：通过，能将 `文枢 AI WriterHub`、`智能文本编辑器`、`Markdown 实时编辑与预览` 等术语映射到英文表达。
- `GET http://127.0.0.1:5173`：通过，返回 HTTP 200。

## 当前限制

- 当前翻译是 deterministic mock 服务，不是真实大模型翻译。
- 中英互译只覆盖内置术语和基础文本转换，质量不代表最终效果。
- 导出译文目前复用复制动作，真实文件导出将在后续阶段接入。
