# 需求说明

## 产品定位

文枢 AI WriterHub 是一个智能文本编辑器 Web 应用。用户的主要任务是在编辑器中生产和处理文档，AI 能力应嵌入编辑、翻译、格式整理和文档问答流程，而不是以聊天窗口作为唯一入口。

## 1. 文本 / Markdown 编辑

### 用户目标

用户可以像普通文本编辑器一样输入、修改、复制内容，同时获得 Markdown 预览和 AI 辅助能力。

### 功能需求

- 支持多行文本输入和编辑。
- 自动检测当前内容是否包含 Markdown 语法。
- 支持三种工作视图：
  - 编辑视图：只显示文本编辑区。
  - 预览视图：只显示 Markdown 渲染结果。
  - 分屏视图：左侧编辑，右侧实时预览。
- 支持常见 Markdown 元素：
  - 标题
  - 粗体 / 斜体
  - 列表
  - 引用
  - 代码块
  - 链接
  - 图片
  - 表格
- 支持复制内容：
  - 复制 Markdown 原文
  - 复制普通文本
  - 复制富文本
- 编辑页右侧提供 AI 助手入口：
  - 优化文章结构
  - 改写润色内容
  - 总结文章要点
  - 翻译为其他语言
  - 解答文档疑问

## 2. 中英互译

### 用户目标

用户可以对编辑器中的内容或粘贴文本进行中英互译，并用适合校对的方式查看原文和译文。

### 功能需求

- 支持中文转英文。
- 支持英文转中文。
- 支持切换翻译方向。
- 支持开始翻译、清空文本、复制译文、导出译文。
- 支持三种展示方式：
  - 分屏对照：左侧原文，右侧译文。
  - 段落交替：按段落交替展示原文和译文。
  - 句对句对照：按句子粒度展示原文和译文。
- 右侧提供翻译增强选项：
  - 学术风格
  - 商务风格
  - 口语自然
  - 术语统一
  - 保留专有名词
- 支持术语表 / 记忆库，用于维护固定翻译规则。

## 3. 一句话整理格式 + 导出

### 用户目标

用户用自然语言描述想要的排版格式，系统自动理解并生成结构化格式配置，用户确认后应用到文档并导出。

### 功能需求

- 支持自然语言格式输入，例如：
  - 正文宋体小四，1.5 倍行距，首行缩进两字符。
  - 标题黑体三号，居中，加页眉页脚。
- 支持智能解析格式要求。
- 解析结果应转为结构化配置：
  - 字体
  - 字号
  - 行距
  - 段落缩进
  - 对齐方式
  - 段前 / 段后
  - 标题样式
  - 页边距
  - 页眉
  - 页脚
  - 页码
  - 纸张方向和大小
- 支持预览排版效果。
- 支持格式化范围：
  - 全文档
  - 选中范围
- 支持导出：
  - Word `.docx`
  - PDF `.pdf`
- 支持查看格式变更记录和格式建议。

## 4. 多文档 AI 问答

### 用户目标

用户上传多个文档后，可以基于文档内容提问，并看到回答依据来自哪些文件或片段。

### 功能需求

- 支持上传多个文档。
- 支持文档类型：
  - PDF
  - Word
  - Markdown
  - Excel
- 展示文档解析状态：
  - 已解析
  - 建立索引中
  - 可检索
  - 解析失败
- 支持选择参与问答的文档。
- 支持按相关度展示检索结果。
- 支持对话式问答。
- 回答中展示来源引用：
  - 来源文件名
  - 页码或位置
  - 引用片段
  - 相关度
- 支持查看引用原文。
- 支持复制回答和重新生成回答。

## 非目标

- 当前阶段不实现完整后端。
- 当前阶段不实现真实 AI 调用。
- 当前阶段不创建可运行前端应用。
- 当前阶段只沉淀项目需求、UI 分析、实现路线和 coding 日志。
