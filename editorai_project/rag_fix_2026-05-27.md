# RAG 修复记录 - 2026-05-27

本记录用于接续 `next_context_current.md`。原上下文文件存在历史乱码，直接大段改写风险较高，因此本次新增独立记录文件。

## 本次问题

用户在知识库页面点击失败文档重试解析，前端提示：

```text
解析失败：{"detail":"文档内容为空，无法生成知识库索引"}
```

实际文档 `关于组织学生选课（预选）的通知` 并不为空，数据库中长度约为 1001 字。

## 根因

后端 RAG 切片函数 `backend/app/services/rag_service.py::split_document_chunks` 有逻辑漏洞：

- 如果第一段文本一开始就超过 `TARGET_CHUNK_SIZE`；
- 且 `current_chunk` 仍为空；
- 旧代码不会把该超长 block 放进 `current_chunk`；
- 导致这一整段被跳过，最终 chunks 为空；
- 后端误判为“文档内容为空”。

## 修复

修改文件：

- `backend/app/services/rag_service.py`

修复点：

```python
if current_chunk:
    chunks.append(current_chunk)
    prefix = overlap_prefix(current_chunk)
    current_chunk = f"{prefix}\n\n{block}".strip() if prefix else block
else:
    current_chunk = block
```

这样当首段就是长段时，会先把 block 放进 `current_chunk`，再进入后续长文本切片循环。
